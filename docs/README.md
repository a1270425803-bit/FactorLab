# FactorLab — AI 驱动的 A 股量化因子工厂

> **一句话介绍**：让 AI 自动编写、验证、评分、回测量化因子，人类只需要设定研究方向。

---

## 一、这是什么？

FactorLab 是一套**全自动量化因子挖掘系统**。它的核心理念是：

```
人类设定研究方向 → AI 生成因子代码 → 安全沙箱执行 → 8 维量化评分 → 冲击成本回测 → 4 维稳健性检验 → 自动入库
```

你告诉它"我要找反转类的因子"，它就会自动生成候选代码，在 1482 只 A 股、15 年历史上跑回测，通过的自动存入数据库。跑完 50 轮后，自动生成一份包含图表的 HTML 研究报告。

**目标用户**：有投资想法但不会写代码的个人投资者，或者想批量验证因子想法的量化研究员。

## 二、核心能力一览

### 2.1 AI 自动生成因子代码

- 接入 DeepSeek API，根据你设定的研究方向自动生成 Python 因子代码
- 每轮自动生成 3 次（最多），选取第一个通过安全检查的版本
- 代码限制在 20 行以内，确保简单可解释
- 支持的研究方向：反转类、动量、量价背离、波动率、行为类、价格路径、时间结构等 12 个方向

### 2.2 AST 安全沙箱

- 纯语法树（AST）级别的代码安全检查，非正则匹配
- 4 类拦截：未来数据（shift 负数）、危险函数调用（eval/open）、逐行循环（禁止 for 遍历股票）、危险模块导入（os/sys）
- 白名单验证 54 条合法代码，**零误报**
- 沙箱超时 30 秒，异常隔离，不会搞崩主程序

### 2.3 8 维量化评分体系

| 维度 | 说明 | 阈值 |
|------|------|------|
| IC 值 | T+5 Spearman 秩相关系数 | > 0.035 |
| IR 值 | IC 均值 / IC 标准差 | > 0.20 |
| 覆盖率 | 非空因子值股票占比 | > 60% |
| 相关性 | 与已入库因子的最大 Spearman 相关 | < 0.70 |
| 日换手率 | 因子值日变化率 | < 1.0 |
| 方向正确性 | Top 30% 股票收益为正的比例 | > 0.55 |
| 5 日秩自相关 | 因子排名 5 天稳定性 | > 0.5 |
| IC 衰减比 | T+5 IC / T+1 IC | > 0.5 |

8 维必须**全部达标**才入库，不是打分制，是关口制。

### 2.4 冲击成本回测引擎

- **不是**简单扣 5‰ 固定成本，而是**逐日逐股计算真实冲击成本**
- 模型：`min(持仓金额 / 日均成交额 × 150%, 2%)`
- 大盘股（日成交 200 亿）冲击 < 0.05%，小盘股（日成交 3 亿）冲击 > 0.5%
- 输出：年化收益、最大回撤、夏普比率、调仓胜率、**分层收益 L1-L5**

### 2.5 4 维稳健性检验

| 维度 | 说明 | 标准 |
|------|------|------|
| 单调性 | 因子值从低到高，收益是否单调递增 | Spearman > 0.3 |
| 样本外稳定性 | 训练集(2010-2018) vs 测试集(2019-2025) IC t 检验 | \|t\| > 2 且测试集 IC > 0.02 |
| IC 衰减 | T+1/T+5/T+10/T+20 IC 衰减速度 | T+20/T+5 > 0.5 |
| 分年验证 | 2020-2025 每年独立 IC | 纯展示，不卡关 |

只有前 3 维全部通过才标记为"稳健"，第 4 维仅作为参考展示。

### 2.6 多样性门控

- 每入库一个新因子，自动与库存中所有已有因子做 Spearman 秩相关
- 相关性 > 0.70 直接拒绝（"跟已有的太像了"）
- 确保因子库中的因子是真正**多样化**的

### 2.7 全自动批量运行

- 一键启动 50 轮全自动因子挖掘
- **双层熔断机制**：
  - 单轮 3 次代码生成均失败 → 跳过本轮
  - 连续 10 轮无因子入库 → 自动暂停，生成诊断报告
- **断点续跑**：中断后 `--resume` 从上次停止处继续
- **Dry-run 模式**：不消耗 API 额度的模拟运行，用于调试
- 50 轮 API 成本预算 ≤ ¥6.00（DeepSeek 定价）

### 2.8 多因子组合合成（ICIR 加权）

- 自动读取全部入库因子，按 ICIR（Information Coefficient / Information Ratio）加权合成
- 负 ICIR 因子权重自动归零
- 组合因子 vs 最佳单因子对比（目标：组合夏普 ≥ 单因子 80%）
- **仅展示不入库**——不干扰正常因子库

### 2.9 静态 HTML 研究报告

跑完批量后，自动生成一份**可双击打开的 HTML 报告**，包含 7 个板块：

| 板块 | 内容 |
|------|------|
| **a. 因子库总览** | 所有入库因子的 IC/IR/夏普/回撤/单调性/样本外状态表格 |
| **b. IR 分布图** | 因子信息比率柱状图，颜色标注优秀/一般/差 |
| **c. 回测指标** | Top 10 因子夏普排名 + 各指标对比表 |
| **d. 成本仪表盘** | 累计 API 成本趋势图 + 单轮成本分布 |
| **e. 方向对比** | 各研究方向平均夏普比率条形图 |
| **f. ICIR 合成** | 多因子组合 vs 最佳单因子对比 |
| **g. 分年验证** | 三时段入库统计，2020-2025 重点高亮 |

- 图表使用 matplotlib 生成 → **base64 内嵌 PNG**，无需网络
- CSS 全部内嵌，不引用外部文件
- 单文件 < 5MB，可直接发微信/邮件

### 2.10 自然语言因子检索

- 输入"有哪些量价背离的因子？"→ 自动分词匹配
- 支持结构化过滤："夏普 > 1.0 且 IC > 0.03 的反转类因子"
- **参数化查询**防止 SQL 注入，白名单过滤条件

## 六、安全设计

这是项目的核心差异化之一——不是"能用就行"，而是有完整的**多层安全体系**：

| 层级 | 机制 | 说明 |
|------|------|------|
| **代码安全** | AST 语法树审查 | 4 类危险模式拦截，白名单验证 54 条合法代码，零误报 |
| **执行隔离** | 沙箱超时 30s | 异常不会搞崩主程序 |
| **权力分割** | 人类锁死 program.md 前三章 + score.py | AI 只能写建议，不能自动修改研究规程 |
| **启动保护** | MD5 校验 | score.py 被修改则拒绝启动 |
| **熔断机制** | 双层熔断 | 单轮 3 次 + 中期 10 轮 |
| **防注入** | 参数化 SQL | 100% ? 占位符 + 白名单 |
| **成本控制** | 预算上限 ¥6.00/50 轮 | DeepSeek 极低 API 成本 |
